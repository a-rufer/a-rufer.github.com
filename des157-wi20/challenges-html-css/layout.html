Structure, Phrasing, and Display

Structural (block) and phrasing (inline) elements differ in several ways. The purpose of structural elements are based more on giving the page structure and giving the element structural meaning, while phrasing elements are more focused on styling. Structural elements take up the entire width and are on their own, new line, while phrasing elements display on the same line.

The CSS display property determines the display behavior of an element and some common values include the following:

  • inline: display as inline element
  • block: display as block element
  • contents: makes the container disappear
  • flex: display as block-level flex container
  • grid: display as block-level grid container
  • inline-block: display as inline-level clock container
  • none: removes the element
  • initial: sets to default value
  • inherit: inherits the property from the parent

So, by setting the display property of an element, you can change the default display of the elements and thus have more flexibility abd creativity in the layout of the page. For example, by declaring a { display:block; }, link elements can be formatted with height and width, and be able to be clicked on in that entire area rather than just the text.

Box Model

The CSS box model is a model that governs the layout of HTML elements and how they are displayed. Essentially, each element is contained in a box. From inside to the outside, the box consists of the content (which contains the content of the element) , padding, border, and margin. The thickness of these parts can be set to specific values that affect the display. With contiguous elements, margins may collapse, with the space between the elements only being as wide as whichever element had a larger margin. The box-sizing property is set to context-box by default, meaning that the width and height of elements (specified with width and height attributes) refers to the width and height of the content box. However, it can be set to padding-box, border-box, or margin-box.

Background Images

Images in <img> tags are used when the image is an important piece of content. It allows it to have an alt attribute. Background images are not part of the content and have properties that make it suited to filling an element, so it can go in the background. Background images are done in CSS, while <img> tags are done in HTML, making them load at different times.
