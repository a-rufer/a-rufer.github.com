About Media Queries

Media queries are a way of detecting and responding to screen size and media types through applying the corresponding CSS rules for that screen size or media type. This makes media queries a vital component of responsive design.

The general syntax for a media query is
@media not|only mediatype and (mediafeature and|or|not mediafeature)
{ /* CSS styles here */ }

For example:
@media only screen and (min-width: 600px)
{ /* CSS styles here */ }

Media queries can be either min-width or max-width, which function in opposite ways. Min-width checks that the screen has a minimum of the given width, and applies its rules to all screens that have at least that width. Max-width checks if the screen is at most the given width, and applies its rules when the screen is less than or equal to the given width.

As a general rule, we do mobile-first design, meaning we develop for small screens first and work our way up to larger layouts. This is because progressive enhancement of a mobile design is much simpler, easier, and cleaner than progressive degradation of a layout for large screen. In other words, it is easier to build up to more complex designs from simpler designs than it is to remove layout and features from complex designs.

Relative Units

In web design, we use relative units because they are convenient for responsive design. They also eliminate the need for the developer to do more calculations themself and allows for automatic adjustment based on changes in other parts of CSS, instead of needing to go through the code and change the sizes of all affected elements. For example, setting an element's width to a percentage allows the element to resize based on the size of the screen or parent elements. The "em" unit is another relative unit and refers to the font size of the element, which is helpful is keeping sizes proportional to the font without needing to calculate the corresponding size and applying an additional size, since it automatically adjusts to changes. "vw" and "vh" are relative to 1% of the width and height of the screen, respectively.

Meta Viewport Tag

The meta viewport element signals to the browser how the page size should be determined, which is important for responsive design. The attribute content has the value "width=device-width", which tells the browser to make the page the width of the screen or window, and "initial-scale=1", which tells the browser that the zoom should equal 1.
