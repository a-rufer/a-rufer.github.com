Inheritance in CSS

In CSS, inheritance is how styles of parent elements are passed on to their child elements. This is useful because inheriting properties allows a property to be defined once on a parent element, and not needed to be defined in every child element individually. Inheritance does not apply to all properties, however. Properties such as color, font-family, font-size, and line-height are inherited, but properities such as text-align, margin, padding, and border are not inherited.

Specificity in CSS

In CSS, specificity helps determine which CSS rules is applied to an element, especially since multiple contradictory rules may target an element. Essentially, rules with more specific rules override less specific ones. The specificity hierarchy is as follows: inline styles; IDs; classes, attributes, and pseudo-classes; and finally elements and pseudo-elements. Specificity is calculated by adding 1000 for an inline style, 100 for each ID, 10 for each attribute, class, or pseudo-class, and 1 for each element or pseudo-element name. If selectors have equal specificity, the last applied rule applies. A compound selector is a selector that combines selectors, while a multiple selector is a way of groups selector such that they all have the same declaration, allowing you to apply the same style to different elements with minimal repetition.

ID and Class Attributes

ID and class attributes are two different attributes that are helpful in grouping elements for styling. When using an ID attribute as a selector in CSS, it is prefixed with '#'. Class attributes are prefixed with ".". ID attributes have higher specificity than class attributes. In addition, IDs can only apply to one element, while the same class can apply to multiple elements.
